DNS Resolution

The Domain Name System (DNS) is like the phonebook of the internet. When users type domain names like google.com, DNS helps translate these names into IP addresses which computers use to identify each other on the network.

DNS Resolution Steps:

  1. User types a domain name in the browser (e.g., google.com).
  2. Browser checks local DNS cache.
  3. If not found, request is sent to DNS Recursive Resolver.
  4. Resolver queries Root DNS server for TLD (Top Level Domain) info.
  5. Root server responds with the TLD server address (e.g., .com server).
  6. Resolver queries the TLD server for domain-specific name server.
  7. TLD server responds with Authoritative Name Server address.
  8. Resolver queries Authoritative Name Server to get the IP address.
  9. Resolver returns IP to browser, which connects to the web server.
